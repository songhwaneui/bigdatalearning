# bigdatalearning - 회의록

### 2019.07.26
```
회의 내용
1. 다음주부터 딥러닝, 자연어 처리과정이 들어가지만, 데이터 수집을 진행해야함.
-> 주말동안 맘카페 데이터를 일부분 크롤링하여 실습이나 복습 자료로 써볼 수 있음.
-> 크롤링 봇은 자동화 시키는 대신에 만약 데이터 수집하는 형태(예.. 정규식)가 변할 것을 대비하여 일 부분을 쉽게 수정할 수 있게 구현해야함.

2. 분석을 나중에 진행한다해도 현재 단계에서 서비스에 대한 구체화 계획이 필요할 거 같음.
-> 주말을 이용하여 실제 서비스를 하는 페이지라고 생각하고 전체 흐름도 각자 그려옴. (웹 프로토타입이나, 직직접 필기로)

3. 백엔드 프론트엔드에 대해 정확한 역할 배분 필요
-> 이 부분에 대해서는 질문을 하려고 함.


보완 사항
1. 시스템 즉 서비스 구체화
2. 발표 자료 디자인에 투자
3. 단어에 대한 감정 분석을 할 시 정확한 오차 범위를 줄여 자연어 처리를 해야함. (즉 기술적인 부분에서도 구체화 시켜야함.)
4. 교육 과정치고는 분석이 차지하는 비용이 적음. (분석할 수 있는 요소를 생각도 해보고, 서비스 부분에 좀 더 연구해야할 필요가 있을 거 같음.)

```
